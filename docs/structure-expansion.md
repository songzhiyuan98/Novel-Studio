# 结构扩容机制（Structure Expansion）

当系统检测到"结构密度不匹配题材"时，会自动触发结构扩容流程，将短剧情升级为长篇成长线。

---

## 一、为什么需要结构扩容？

### 1.1 问题场景

某些题材天然需要长线叙事：

| 题材类型 | 特征 | 需要的结构 |
|----------|------|-----------|
| 重生 + 创业 | 人生重来型 | 多阶段成长线 |
| 校园 + 多女主 | 关系慢热 | 长周期铺垫 |
| 修仙 + 势力 | 境界跃迁 | 阶段性目标 |
| 都市 + 商战 | 逐步扩张 | 圈层升级 |

如果不扩容，会出现：
- 很快写完"小故事" → 剧情枯竭
- 女主线来不及铺垫 → 感情显得工具人
- 世界观设定浪费 → 格局过小

### 1.2 Evaluator 检测机制

Evaluator 会评估结构密度：

```
┌─────────────────────────────────────────────────────────────┐
│                    结构密度评估                              │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  题材规模：重生 + 创业 + 校园 + 多女主                        │
│  ↓                                                          │
│  需要结构：多阶段成长线（至少 3 阶段）                        │
│                                                              │
│  当前结构：单线（搞钱 → 父亲限制 → 校园项目）                 │
│  ↓                                                          │
│  评估结论：❌ 过短，只够 20 章                               │
│                                                              │
│  ⚠️ 触发结构扩容                                            │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## 二、三阶段人生主线模型

对于"人生重来型"题材，标准扩容为三阶段：

### 🧱 第一阶段：校园生存 & 第一桶金

| 项目 | 内容 |
|------|------|
| 章节范围 | 第 1-80 章 |
| 关键词 | 试错、低成本试水、关系建立 |
| 主线目标 | 从"有背景但没话语权" → "有第一笔真实成就" |
| 成长主题 | 从"穷人思维" → "资源整合思维" |
| 关系进展 | 认识核心人物（女主1+女主2+未来合伙人） |

**典型节点**：
- 引爆点：发现富二代身份但钱不在自己手里
- 第一次代价：快速套利失败/被父亲否定
- 中段反转：校园项目小成功
- 最黑暗时刻：项目崩盘/被误解
- 阶段终局：证明自己不是废物

---

### 🧱 第二阶段：身份跃迁 & 圈层升级

| 项目 | 内容 |
|------|------|
| 章节范围 | 第 80-200 章 |
| 关键词 | 走出校园、真实创业、社会关系 |
| 主线目标 | 从"学生创业者" → "真正的操盘者" |
| 成长主题 | 从"证明自己" → "建立体系" |
| 关系进展 | 关系进入"选择与代价"阶段 |

**典型节点**：
- 引爆点：项目从校园扩展到城市
- 代价：家庭线冲突升级（父亲 vs 主角路线分歧）
- 中段反转：获得外部认可/投资
- 最黑暗时刻：第一次真正失败（大坑）
- 阶段终局：建立自己的团队和资源

---

### 🧱 第三阶段：格局扩张 & 人生选择

| 项目 | 内容 |
|------|------|
| 章节范围 | 200+ 章 |
| 关键词 | 格局、代价、情感终局 |
| 主线目标 | 从"想补遗憾的人" → "能决定他人命运的人" |
| 成长主题 | 创业不再只是赚钱，而是"选择成为什么样的人" |
| 关系进展 | 多女主关系线进入终局选择（慢热兑现） |

**典型节点**：
- 引爆点：进入更大的舞台
- 代价：富二代身份的真正意义（继承 vs 自建）
- 反转：揭示更大的真相/机会
- 最黑暗时刻：核心价值观的考验
- 终局：人生选择与关系终局

## 三、扩容流程

### 3.1 检测触发

```
用户输入题材/简介
        │
        ▼
┌───────────────────┐
│    Evaluator      │
│  评估结构密度      │
└─────────┬─────────┘
          │
    ┌─────┴─────┐
    ▼           ▼
 密度匹配     密度不匹配
    │           │
    ▼           ▼
 正常流程   触发扩容流程
```

### 3.2 扩容执行

```
触发扩容
    │
    ▼
┌───────────────────┐
│  Plot Architect   │
│  升级为多阶段主线  │
└─────────┬─────────┘
    │
    ▼
┌───────────────────┐
│  Producer 汇总    │
│  生成结构卡片      │
└─────────┬─────────┘
    │
    ▼
┌───────────────────┐
│  最小问题          │
│  确认主轴偏好      │
└─────────┬─────────┘
    │
    ▼
┌───────────────────┐
│  更新 Outline     │
│  继续正常流程      │
└───────────────────┘
```

### 3.3 用户确认点

扩容后只问一个核心问题：

```
┌─────────────────────────────────────────────────────────────┐
│                                                              │
│  这本书不适合"短剧情密集爆点流"，                             │
│  更适合"阶段成长流"。                                        │
│  我已将结构扩容为三阶段主线。                                 │
│                                                              │
│  你更想强化哪条线作为长期爽点主轴？                           │
│                                                              │
│  ○ A. 创业成长爽（项目升级、商战、资源博弈）                  │
│  ○ B. 人生逆袭爽（身份、圈层跃迁、家庭博弈）                  │
│  ○ C. 情感推进爽（多女主慢热关系变化）                        │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

选择后，系统会把"剧情空间"往对应线上加厚。

## 四、世界观分层机制

对于复杂世界观（宇宙流/纪元无限流），采用分层防爆炸：

### 4.1 三层世界观

| 层级 | 名称 | 说明 | 要求 |
|------|------|------|------|
| L0 | 本章可见规则 | 当前章节涉及的规则 | 必须清晰可验证 |
| L1 | 本卷解释真相 | 本卷会揭示的真相 | 有轮廓即可 |
| L2 | 终局宇宙观 | 终局会揭示的真相 | 只留占位符，不细化 |

### 4.2 可写阈值

| 阈值 | 说明 | 最低要求 |
|------|------|----------|
| Threshold A | 能写前 3 章 | 最小规则集 6 条 + 引爆点 |
| Threshold B | 能写一卷 20-30 章 | 对手体系 + 反转节点 + 扩展计划 |

### 4.3 渐进细化原则

```
开始写作
    │
    ├── L0 规则必须完整
    │
    ├── L1 规则有轮廓
    │
    └── L2 规则只占位
           │
           │ 随着写作推进
           ▼
    L1 逐步细化 → 变成 L0
    L2 保持占位 → 直到终章前
```

## 六、阶段过渡设计

### 6.1 阶段过渡检测

```
当前进度: 第 75 章
当前阶段: 第一阶段（校园生存）
阶段目标完成度: 85%
        │
        ▼
┌───────────────────┐
│  过渡检测         │
│  是否准备升阶段？  │
└─────────┬─────────┘
          │
    ┌─────┴─────┐
    ▼           ▼
  是            否
    │           │
    ▼           ▼
准备过渡章节   继续当前阶段
```

### 6.2 过渡章节设计

阶段过渡需要专门设计的"过渡章节"：

| 功能 | 说明 |
|------|------|
| 收尾 | 当前阶段主线收尾 |
| 回顾 | 回顾成长与变化 |
| 伏笔回收 | 收回本阶段主要伏笔 |
| 新挑战预告 | 预告下一阶段的挑战 |
| 格局提升 | 展示新的舞台 |
