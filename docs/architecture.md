# 系统架构（System Architecture）

本文档描述 Novel Studio 的核心组件与高层架构设计。

---

## 一、系统目标

把"写小说"变成一个可交互的编剧室工作流：

1. **用户不需要会写剧情**，只需要表达偏好 + 做判断 + 确认进入正史
2. **系统能长期连载不崩**：设定一致、人物不崩、节奏不崩、可回滚

---

## 二、设计原则（硬规则）

| 原则 | 英文名 | 说明 |
|------|--------|------|
| 正史闸门 | Canon Gate | 只有用户 Confirm 的内容才进入正史（Story Bible / 已确认章节事实） |
| 上下文隔离 | Context Isolation | 子 Agent 永远只看"简报包"，不看完整聊天历史 |
| 按需子 Agent | Task-on-demand | 除了少数常驻角色，大部分 Agent 任务制拉起 |
| 分流决策 | Triage Router | 遇到冲突/缺口，先分流：内部解决 vs 询问用户 vs provisional 占位继续写 |
| 工件驱动 | Artifact-first | 章节、设定、大纲、角色卡都是可编辑工件（Draft/Reviewed/Confirmed/Locked） |
| 质量闸门 | Quality Gate | 正文发布前必须过编辑+一致性检查（至少 Reviewed） |

## 四、组件职责详解

### 4.1 Producer Orchestrator（总控）

系统的"大脑"，唯一与用户直接交互的组件：

| 职责 | 说明 |
|------|------|
| 需求解析 | 从用户自由表达中提取：硬约束、软偏好、禁区、未知项 |
| 任务派发 | 创建任务并分配给子 Agent |
| 产出整合 | 汇总子 Agent 产出成用户可判断的内容 |
| 冲突仲裁 | 通过 Triage Router 决定：内部解决 / 询问用户 / 占位继续 |
| 版本管理 | 管理工件状态机、版本控制、回滚 |
| Canon 写入 | 只有 Producer 有权写入正史 |

### 4.2 Agent Runtime（子 Agent 运行时）

按需创建任务执行器：

- **Divergent**：发散方案 A/B/C
- **Worldbuilder**：设定/修炼体系/规则最小集
- **Plot Architect**：结构节点/反转设计
- **Scene Designer**：分镜/场景卡
- **Character/Romance**：女主与关系推进
- **Research**：外部资料调研（禁止改 Canon）

### 4.3 Context Router（上下文路由）

给每个 Agent 生成 Context Packet（简报包），确保：

- Agent 不读取完整聊天历史
- Agent 只获得与任务相关的最小上下文
- 防止"灵感污染"

### 4.4 Quality Gate（质量闸门）

- **Editor**：节奏/爽点/文风评估
- **Continuity**：设定一致性检查
- 评分与阻断机制

### 4.5 Artifact Store（工件存储）

存储所有可编辑工件：

- 章节（Chapter）
- 设定圣经（Story Bible）
- 大纲（Outline）
- 角色卡（Character Cards）
- 时间线（Timeline）
- 伏笔表（Promises）
- 审查报告（QA Reports）

### 4.6 Versioning（版本控制）

- 工件版本追踪
- 修订历史
- 回滚能力
- 分支（可选，用于"假如"场景）

## 六、工件状态机

所有工件遵循统一的状态流转：

```
Draft ──(QA审核)──► Reviewed ──(用户确认)──► Confirmed ──(冻结)──► Locked
  │                    │                        │
  │◄────(打回修改)─────┘                        │
  │                                             │
  │◄───────────(创建Revision回滚)───────────────┘
```

| 状态 | 说明 | 可操作 |
|------|------|--------|
| Draft | 草稿，可随意修改 | Edit/Rewrite/Delete |
| Reviewed | 通过 QA 初审 | Edit/Confirm |
| Confirmed | 用户确认，进入 Canon | 只读（改动需开 Revision） |
| Locked | 冻结，关键设定 | 只读 |

## 七、与传统 AI 写作的区别

| 维度 | 传统 AI 写作 | Novel Studio |
|------|-------------|--------------|
| 剧情来源 | Prompt 即兴生成 | 世界模拟 + 结构规划 |
| 用户角色 | 提词人 | 决策者 + 审稿人 |
| 上下文 | 全量历史 | 隔离简报包 |
| 一致性 | 靠运气 | 结构化守护 |
| 长篇能力 | 极弱 | 原生支持 |
| 可回滚 | 无 | 版本控制 |
| 质量保障 | 无 | QA 闸门 |
