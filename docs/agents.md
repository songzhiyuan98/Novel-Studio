# Agent 编制与职责（Agent Roster）

本系统采用 **常驻 + 按需** 的 Agent 编制模式：
- 常驻 Agent：MVP 必需，始终存在
- 按需 Agent：任务制拉起，完成即释放

---

## 一、常驻 Agent（MVP 必需）

### 1. Producer（总控）

**定位**：系统的"大脑"，唯一与用户直接交互的 Agent。

| 职责 | 说明 |
|------|------|
| 需求解析 | 从用户自由表达中提取：硬约束、软偏好、禁区、未知项 |
| 任务派发 | 创建任务并分配给子 Agent |
| 产出整合 | 汇总子 Agent 产出成用户可判断的内容 |
| 冲突仲裁 | 通过 Triage Router 决定处理路径 |
| 版本管理 | 管理工件状态机、版本控制 |
| Canon 写入 | **唯一有权写入正史的角色** |

**能力边界**：
- ✅ 可以：解析、派单、整合、仲裁、写入 Canon
- ❌ 不可以：直接生成正文、直接创造设定

---

### 2. Evaluator（评估）

**定位**：材料充足性评估器，决定"能不能开写"。

| 职责 | 说明 |
|------|------|
| 可写性评估 | 判断材料是否达到可写阈值 |
| 缺口识别 | 找出缺失的关键信息 |
| 最小问题集 | 生成 2-3 个选择题以补齐缺口 |
| Coverage Map | 输出材料覆盖地图 |

**可写阈值**：

| 阈值 | 说明 | 最低要求 |
|------|------|----------|
| Threshold A | 能写前 3 章 | 最小规则集 6 条 + 引爆点 |
| Threshold B | 能写一卷 20-30 章 | 对手体系 + 反转节点 + 扩展计划 |

**能力边界**：
- ✅ 可以：评估、识别缺口、生成问题
- ❌ 不可以：创造内容、做决策

---

### 3. Writer（正文）

**定位**：严格按场景卡写正文的执行者。

| 职责 | 说明 |
|------|------|
| 正文生成 | 根据场景卡生成章节草稿 |
| 风格遵守 | 按 Style Guide 写作 |
| 约束遵守 | 严格遵守禁止项 |

**输入（Context Packet）**：
- Canon Snapshot
- Recent Summary（最近 2-3 章）
- SceneCards
- Constraints（禁止项）
- Style Guide

**能力边界**：
- ✅ 可以：写正文、润色表达
- ❌ 不可以：自创世界规则、改变设定、偏离场景卡

---

### 4. QA（Editor + Continuity 合体）

**定位**：质量守门员，包含编辑与一致性检查两个功能。

#### Editor 功能

| 检查项 | 说明 |
|--------|------|
| 节奏 | 是否拖沓/太赶 |
| 爽点密度 | 是否有钩子/满足感 |
| 文风 | 是否符合 Style Guide |
| 可读性 | 是否流畅/有代入感 |

#### Continuity 功能

| 检查项 | 说明 |
|--------|------|
| 设定一致性 | 是否违反 Bible |
| 人物一致性 | 角色行为是否符合人设 |
| 时间线一致性 | 事件顺序是否正确 |
| 伏笔追踪 | 已埋伏笔是否有回收计划 |

**输出**：
- 评分（0-100）
- 问题列表
- 修改建议
- 阻断/通过决定

**能力边界**：
- ✅ 可以：评估、打分、提出问题、建议修改
- ❌ 不可以：直接修改内容

---

## 二、按需 Agent（任务制）

### 1. Divergent（发散）

**定位**：创意发散器，产出多个方案供选择。

| 职责 | 说明 |
|------|------|
| 方案发散 | 产出 A/B/C 主线方案 |
| 创意探索 | 探索不同的故事可能性 |
| 风险标注 | 标注每个方案的风险与难度 |

**产物**：`Proposals[]`

---

### 2. Worldbuilder（世界构建）

**定位**：世界观与规则系统设计师。

| 职责 | 说明 |
|------|------|
| 规则设计 | 设计世界运行的最小规则集 |
| 体系构建 | 修炼体系/能力体系/经济体系 |
| 限制设计 | 设计合理的限制与代价 |

**产物**：`WorldRules`、`PowerSystem`

**最小规则集示例**（6 条）：
1. 家里有钱 ≠ 主角可支配
2. 父子关系有矛盾
3. 资金获取要过关
4. 时代红利有限制
5. 身份是双刃剑
6. 重生记忆有盲区

---

### 3. Plot Architect（剧情架构）

**定位**：结构设计师，规划关键节点与节奏。

| 职责 | 说明 |
|------|------|
| 节点规划 | 设计引爆点/代价/反转/终局 |
| 节奏设计 | 规划爽点分布与张力曲线 |
| 阶段划分 | 划分卷/阶段/成长线 |

**产物**：`ArcBeats`、`Outline`

---

### 4. Scene Designer（场景设计）

**定位**：分镜师，把章节拆解为可执行场景。

| 职责 | 说明 |
|------|------|
| 场景拆解 | 每章拆成 3-6 个场景 |
| 场景设计 | 设计目的/冲突/钩子 |
| 节奏控制 | 控制场景内部节奏 |

**产物**：`SceneCards[]`

---

### 5. Character & Romance（角色与情感）

**定位**：角色设计师，负责人物塑造与关系推进。

| 职责 | 说明 |
|------|------|
| 角色设计 | 设计人物动机/压力/边界/秘密 |
| 女主池管理 | 设计女主候选与各自目标线 |
| 关系推进 | 规划关系发展节奏 |
| 相遇设计 | 设计相遇事件与化学反应 |

**产物**：`CharacterCards[]`、`RomanceProgress`

---

### 6. Research（调研）

**定位**：外部资料调研员。

| 职责 | 说明 |
|------|------|
| 资料搜集 | 搜集外部参考资料 |
| 事实核查 | 核查时代/行业/专业细节 |
| 素材整理 | 整理可用素材供其他 Agent 使用 |

**能力边界**：
- ✅ 可以：搜集、整理、提供参考
- ❌ 不可以：**禁止改 Canon**

## 四、Agent 通信规则

### 铁律

1. **Agent 之间禁止直接通信**
2. 所有通信必须经过 Producer
3. Agent 只接收 Context Packet，不读完整历史
4. Agent 只输出 Artifact + Issue Packet

### 信息流向

```
Producer ──(Context Packet)──► Agent ──(Artifact + Issue)──► Producer
```

## 五、Agent 数量建议

| 阶段 | 常驻 | 按需 | 总计 |
|------|------|------|------|
| MVP | 4 | 0-2 | 4-6 |
| v1.0 | 4 | 3-4 | 7-8 |
| v2.0 | 4 | 6 | 10 |

> 原则：先做简后做繁，常驻尽量少，按需灵活扩展。
