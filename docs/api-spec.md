# API Specification

> See also: `docs/superpowers/specs/2026-03-25-design-v2.md` for the v2 design spec.
> See also: `docs/superpowers/specs/2026-03-24-core-architecture-design.md` for the original design spec.

All pipeline tasks flow through: User -> Chat Agent -> Orchestrator -> Worker. There are no direct worker-invocation endpoints.

## External API (Web to Backend)

### Project Templates

#### POST /project-templates
Create a new project template. Typically generated by Planner based on genre, then confirmed by user.

Request:
```json
{
  "genre": { "primary": "string", "tags": ["string"] },
  "format": {
    "chapter_length": { "min": 2000, "max": 3000, "unit": "chars" },
    "volume_size": 100,
    "language": "zh-CN"
  },
  "character_dimensions": [
    { "key": "string", "label": "string", "type": "string|number|list" }
  ],
  "relationship_dimensions": [
    { "key": "string", "label": "string", "type": "string|number" }
  ],
  "style_profile": {
    "base": {
      "pov": "first|third_limited|third_omniscient",
      "tense": "past|present",
      "prose_density": "light|medium|dense"
    },
    "custom": [
      { "key": "string", "value": "string" }
    ]
  },
  "writing_rules": ["string"],
  "qa_custom_dimensions": ["string"]
}
```

#### GET /project-templates/:templateId
Get template detail.

#### PUT /project-templates/:templateId
Update template. Changes to dimensions may trigger impact analysis on existing project data.

#### DELETE /project-templates/:templateId
Delete template (only if no projects reference it).

### Project Management

#### POST /projects
Create project. References a ProjectTemplate instead of hardcoding genre-specific fields.

Request:
```json
{
  "title": "string",
  "description": "string",
  "template_id": "string"
}
```

Note: Genre, style profile, chapter length, volume size, and all genre-specific schemas are derived from the referenced ProjectTemplate.

#### GET /projects/:projectId
Fetch project summary, including resolved template fields.

#### PATCH /projects/:projectId
Update project metadata (title, description, template_id).

### Model Configuration

#### GET /projects/:projectId/model-config
Get current model configuration.

#### PUT /projects/:projectId/model-config
Set model configuration.

Request (simple mode):
```json
{
  "config_mode": "simple",
  "provider": "openai|anthropic|google|deepseek",
  "api_key": "string"
}
```

Request (advanced mode):
```json
{
  "config_mode": "advanced",
  "workers": {
    "chat": { "provider": "string", "model": "string", "api_key": "string" },
    "planner": { "provider": "string", "model": "string", "api_key": "string" },
    "writer": { "provider": "string", "model": "string", "api_key": "string" },
    "qa": { "provider": "string", "model": "string", "api_key": "string" },
    "summarizer": { "provider": "string", "model": "string", "api_key": "string" }
  }
}
```

### Chat (Primary User Interface)

#### POST /projects/:projectId/chat
Send user message. Chat Agent classifies intent and routes accordingly.

**Supports streaming responses** via Vercel AI SDK. When `stream: true` is set, the response is delivered as a server-sent event (SSE) stream. Users can pause, stop, or interrupt the stream mid-generation.

Request:
```json
{
  "message": "string",
  "stream": true
}
```

Response (non-streaming):
```json
{
  "assistant_message": "string",
  "intent": "casual|canon_edit|pipeline_task",
  "created_artifacts": ["artifact_id"],
  "pending_confirmations": ["artifact_id"],
  "trace": {
    "steps": [
      {
        "timestamp": "ISO8601",
        "actor": "chat_agent|orchestrator|planner|writer|qa|summarizer",
        "action": "string",
        "details": {},
        "tokens": { "input": 0, "output": 0 },
        "duration_ms": 0,
        "cost_usd": 0.0
      }
    ]
  },
  "suggested_actions": ["string"]
}
```

Response (streaming): SSE stream with `data:` events containing partial `assistant_message` chunks, followed by a final event with the full structured response (intent, artifacts, trace, etc.).

#### GET /projects/:projectId/chat/history
Fetch conversation history with pagination.

Query params: `limit`, `before` (cursor).

### Artifacts

#### GET /projects/:projectId/artifacts
Query artifacts by type/status.

Query params: `type`, `status`, `chapter_number`.

#### GET /artifacts/:artifactId
Get artifact detail with content. For chapter_draft artifacts, `scene_segments_json` contains the per-scene breakdown.

#### POST /artifacts/:artifactId/edit
Create new artifact version.

Request:
```json
{
  "content_json": {},
  "edit_note": "string"
}
```

#### POST /artifacts/:artifactId/confirm
Confirm artifact into canon. Triggers canon projection via Orchestrator.

Response:
```json
{
  "confirmed": true,
  "canon_updates": {
    "story_bible_entries": 0,
    "character_states": 0,
    "timeline_events": 0,
    "threads_updated": 0
  },
  "trace": {}
}
```

#### POST /artifacts/:artifactId/reject
Reject artifact. Sets status to rejected.

### Chapters

#### GET /projects/:projectId/chapters
Query chapter timeline with pagination.

Query params: `limit`, `offset`, `status`.

#### GET /chapters/:chapterId
Get chapter detail including latest draft artifact, summary, and scene segments.

#### GET /chapters/:chapterId/summary
Get chapter structured summary (ChapterSummary entity).

#### POST /chapters/:chapterId/scenes/:sceneIndex/rewrite
Rewrite a single scene within a chapter. Only re-generates the specified scene; surrounding scenes are preserved. The Writer receives surrounding scene text for continuity.

Request:
```json
{
  "instructions": "string",
  "updated_blueprint": {},
  "stream": true
}
```

Response: Same structure as POST /chat (supports streaming). The updated scene text replaces the specified scene segment in the chapter artifact.

### Canon Stores (Read-Only Queries)

#### GET /projects/:projectId/canon/characters
Get all current character states. Supports filtering by tier.

Query params: `tier` (core, important, episodic), `status` (active, archived).

#### GET /projects/:projectId/canon/characters/:characterKey
Get specific character state including dynamic dimensions.

#### GET /projects/:projectId/canon/world-rules
Get all confirmed world rules.

#### GET /projects/:projectId/canon/timeline
Get timeline events with pagination.

Query params: `chapter_from`, `chapter_to`, `character`.

#### GET /projects/:projectId/canon/threads
Get unresolved threads.

Query params: `status` (active, resolved, deferred).

#### GET /projects/:projectId/canon/chains
Get development chains.

### Impact Analysis

#### POST /projects/:projectId/impact-analysis
Run impact analysis for a proposed change. Returns affected chapters, conflicting character cards, broken hooks/threads, and severity per conflict.

Request:
```json
{
  "change_type": "rename|attribute_tweak|structural_change",
  "change_description": "string",
  "target_entity": {
    "type": "character|world_rule|story_bible_entry|relationship",
    "id": "string"
  },
  "proposed_change": {}
}
```

Response:
```json
{
  "risk_level": "green|yellow|red",
  "affected_chapters": [
    { "chapter_number": 0, "chapter_id": "string", "conflicts": ["string"] }
  ],
  "affected_characters": [
    { "character_key": "string", "conflicts": ["string"] }
  ],
  "broken_hooks": [
    { "thread_id": "string", "label": "string", "reason": "string" }
  ],
  "resolution_options": [
    {
      "key": "auto_fix|apply_forward|full_rewrite|cancel",
      "label": "string",
      "description": "string",
      "estimated_cost": { "chapters_to_rewrite": 0, "estimated_tokens": 0 }
    }
  ]
}
```

#### POST /projects/:projectId/impact-analysis/:analysisId/resolve
Apply a chosen resolution option from a previous impact analysis.

Request:
```json
{
  "resolution_key": "auto_fix|apply_forward|full_rewrite|cancel"
}
```

### Orchestration Trace

#### GET /projects/:projectId/tasks
Get task history with token usage and cost.

Query params: `limit`, `status`, `worker`.

#### GET /tasks/:taskId
Get task detail including input packet summary and output.

### Issues

#### GET /projects/:projectId/issues
Fetch open issues and evidence.

Query params: `status`, `severity`, `type` (open string, not restricted to a fixed set).

### Audit

#### GET /projects/:projectId/audit
Get audit log.

Query params: `limit`, `event_type`, `actor`, `before`.

## Internal Worker Contracts

All workers are stateless single-shot LLM calls dispatched by the Orchestrator. Workers receive a compiled packet and return structured output. Workers never see chat history.

### Planner Task

Input packet:
- task_objective
- workflow_stage
- current canon (world rules, character states, outline)
- unresolved threads
- development chains
- open questions
- project_template (character_dimensions, relationship_dimensions, writing_rules)
- output_contract

Output schema:
```json
{
  "summary": "string",
  "assumptions": ["string"],
  "questions": ["string"],
  "artifacts": [
    {
      "type": "outline|scene_card|world_rule|character_card|development_chain|evaluation",
      "title": "string",
      "content": {}
    }
  ],
  "issues": [
    {
      "severity": "low|medium|high",
      "type": "string (open, e.g. continuity|motivation|world_logic|power_scaling|...)",
      "description": "string"
    }
  ]
}
```

Note: Scene cards in the output are detailed blueprints including combat design, dialogue specifics, reversal layers, NPC reactions, and forbidden patterns. See v2 design spec section 5 for full blueprint structure.

### Writer Task

Input packet:
- chapter_objective
- scene_cards (detailed blueprints with combat/dialogue/reversal specifics)
- scene_index (optional; if provided, only write this scene — for per-scene rewriting)
- surrounding_scene_text (for per-scene rewriting continuity)
- relevant canon (character states with dynamic dimensions, world rules)
- recent chapter summaries
- relationship states (base_scores + custom_scores)
- style_profile (base + custom from ProjectTemplate)
- format_constraints (from ProjectTemplate.format — chapter_length, etc.)
- writing_rules (from ProjectTemplate)
- output_contract

Output schema:
```json
{
  "chapter_title": "string",
  "chapter_summary": "string",
  "scene_segments": [
    {
      "scene_index": 0,
      "scene_key": "string",
      "text": "string"
    }
  ],
  "chapter_text": "string (concatenated scene_segments)",
  "introduced_provisional_elements": ["string"],
  "risks": ["string"]
}
```

Note: `scene_segments` is the canonical representation. `chapter_text` is provided as a convenience (concatenation of all scene texts). For per-scene rewrites, only the targeted scene segment is returned.

### QA Task

Input packet:
- chapter_draft (with scene_segments)
- relevant canon (character states with dynamic dimensions, world rules)
- recent chapter summaries
- timeline events
- unresolved threads
- project_template (for custom QA dimensions and writing rules)

Output schema:
```json
{
  "decision": "pass|pass_with_notes|revise|block",
  "overall_notes": "string",
  "issues": [
    {
      "severity": "low|medium|high",
      "type": "string (open — continuity|style|motivation|pacing|world_logic|power_scaling|custom...)",
      "scene_index": 0,
      "description": "string",
      "evidence_refs": [
        {
          "source_type": "canon_entry|chapter_summary|character_state|timeline_event",
          "source_id": "string",
          "quote": "string"
        }
      ],
      "suggested_fix": "string"
    }
  ]
}
```

Note: `type` is an open string. Base types are continuity, style, motivation, pacing, world_logic. Additional types may come from `ProjectTemplate.qa_custom_dimensions`.

### Summarizer Task

Input packet:
- chapter_text (full)
- scene_segments
- chapter_number
- character_list (with tiers)
- active_threads

Output schema:
```json
{
  "chapter_number": 0,
  "summary": "string",
  "key_events": ["string"],
  "character_deltas": [
    { "character": "string", "change": "string", "dimension_updates": {} }
  ],
  "new_threads": ["string"],
  "resolved_threads": ["string"],
  "advanced_threads": ["string"],
  "new_world_facts": ["string"],
  "timeline_events": [
    { "event": "string", "location": "string" }
  ],
  "episodic_characters_to_archive": ["string"]
}
```
